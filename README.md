# Challenges in Representation Learning: Facial Expression Recognition Challenge

Kaggle-ის *Challenges in Representation Learning: Facial Expression Recognition Challenge* კომპეტიშენის მიზანია სახის ფოტოების კლასიფიკაცია, ემოციის მიხედვით 7 კატეგორიად - "Angry", "Disgust", "Fear", "Happy", "Sad", "Surprise", "Neutral". 
ავაგე 4 მოდელი, დავიწყე მარტივი არქიტექტურიდან და საბოლოოდ ავაგე რეზნეტის მსგავსი არქიტექტურა.
ყველა ექსპერიმენტის მონაცემები დალოგილია wandb-ზე.
wandb-ს ლინკი: https://wandb.ai/toberi23-free-university-of-tbilisi-/fer2013-emotion

## მიდგომა

- **კონფიგურირებადი მოდელები** - `MLP`, `SmallCNN` და `DeepCNN` კონსტრუქტორშივე იღებენ შემდეგ      არგუმენტებს `activation` / `norm` / `dropout` / `init` (+ `hidden` / `channels`), რაც ამარტივებს სხვადასხვა მიდგომების გატესტვას
- **`fit(build_model, arch, …)`** — ტრეინინგის ფუნქცია. ის ინახავს საუკეთესო ეპოქას, ტრეინინგის დამთავრების შემდგომ ტესტავს ტესტ დატასეტზე, ლოგვას ექსპერიმენტს. არგუმენტად იღებს `lr`, `batch_size`, `weight_decay`, `optimizer_name` და ა.შ. ასეთი არქიტექტურა საშუალებას მაძლევს ბევრი მიდგომა მარტივად გავტესტო.
- **Sanity checks** - ტესტავს მოდელში რამე ბაგი ხომ არ არის გაპარული: 
   
   I - forward-ის ფორმა უნდა იყოს  `(B,7)`,

   II - საწყისი loss ≈ `log(7) ≈ 1.946` 
  
   III - პატარა მოდელის დაზეპირება.

- **wandb-ზე დალოგვა** -  ექსპერიმენტები დაჯგუფებულია არქიტექტურის მიხედვით (`group=arch`);
  თითო study/search იღებს study-ის შესაბამის ტეგს. ყოველ epoch-ზე
  ვლოგავთ `train/val loss`-სს და `acc`-ს, `val/f1_macro`-ს, `gap/acc`-ს და `lr`-ს; ექსპერიმენტის 
  summary ინახავს საუკეთესო ეპოქას, ტესტ მეტრიკებს და confusion მატრიცას. `wandb.watch` ლოგავს 
  gradient/weight ჰისტოგრამებს.
- **ჰიპერპარამეტრების ძიება.** - დაიმპლემეტირებული მაქს `grid_search`  და `random_search` მეთოდები.

## არქიტექტურა 1 — MLP `mlp_baseline.ipynb`

48×48 სურათს ვშლი flatten-ის დახმარებით და ვსწავლობ fully connected classifier-ს
(`2304 → 256 → 7`, **~592k პარამეტრი**). MLP მოდელს არ შეუძლია აღიქვას მეზობელი უჯრები რამეთუ ის 
ფოტოს აღიქვამს როგორც 2304 განზომილებიან ვექტორს. შესაბამისად მოსალოდნელი შედეგია ანტერფიტი.

თუმცა ტრეინინგმა აჩვენა ოვერფიტი - ტრეინიის სიზუსტე დაახლოებით ~0.69-მდე ავიდა, ხოლო ვალიდაცია
~0.41-ზე გაჩერდა. 

| run | best val | test acc | test F1 | train acc |
|---|---|---|---|---|
| MLP baseline | 0.414 | **0.424** | 0.412 | ~0.69 |

## არქიტექტურა 2 — SmallCNN (ერთი კონვოლუციური ფენა) `cnn_baseline.ipynb`

ამ არქიტექტურაში გამოვიყენე ერთი კონვოლუციური ფენა 
  1. Conv2d — 32 ფილტრი → 1×48×48 → 32×48×48
  2. norm → activation — normalize + ReLU
  3. MaxPool2d(2) — 32×48×48 → 32×24×24
  4. flatten → dropout → Linear — 32×24×24 → 18,432 → 7 
პარამეტრების რაოდენობა ~129k — MLP-ზე ნაკლები.

SmallCNN-ის ბეიზლაინ მოდელმა უკეთესი შედეგი აჩვენა ვიდრე MLP-მ. 

| model | params | best val | test acc | test F1 | train acc |
|---|---|---|---|---|---|
| MLP (tuned) | 592k | 0.414 | 0.424 | 0.412 | ~0.69 |
| **SmallCNN (baseline)** | **129k** | **0.473** | **0.468** | 0.423 | ~0.80 |

ერთმა კონვოლუციურმა ლეიერმა `78%`-ით ნაკლები პარამეტრით ტესტზე `4%` ით უკეთესი შედეგი აჩვენა. თუმცა
ასევე აღსანიშნავია რომ უფრო მეტად წავიდა ოვერფიტში, ვალიდაციის პიკს მიაღწია **მე-6** ეპოქაზე.

### Study 3 — ნორმალიზაცია (none vs BatchNorm vs LayerNorm)

| norm | best val | best epoch | test acc | test F1 |
|---|---|---|---|---|
| **none** | **0.473** | 6 | **0.468** | 0.423 |
| layer | 0.461 | 5 | 0.452 | 0.415 |
| batch | 0.456 | 15 | 0.462 | **0.437** |

გავტესტე სხვადასხვა ნორმალიზაცია, და ნორმალიზაციის საერთოდ ჩათიშვამ უკეთესი შედეგი დადო ვიდრე 
BatchNorm-მმა LayerNorm-მმა. ეს შედეგი იმით იახსნება რომ ეს მოდელი არ არის საკმარისად ღრმა რათა
ნორმალიზაციამ იმოქმედოს. თუმცა BatchNorm-მმა გამოიწვია ოვერფიტის შემცირება, ვალიდაციის პიკს მიაღწია
**მე-15** ეპოქაზე.

### SmallCNN-ის tuning

გამოვიყენე `grid_search` და მოვძებნე რიგრიგობით საუკეთესო `lr`, `dropout`, `scedule`, `weight decay`.

| search | პარამეტრი | grid | საუკეთესო | best val | test acc | test F1 |
|---|---|---|---|---|---|---|
| — | baseline | lr 1e-3, no reg | — | 0.473 | 0.468 | 0.423 |
| 1 | **learning rate** | {1e-4, 3e-4, 1e-3, 3e-3, 1e-2} | **3e-4** | 0.474 | 0.474 | 0.430 |
| 2 | **dropout** | {0, 0.3, 0.5, 0.7} | **0.3** | 0.479 | 0.477 | 0.437 |
| 3 | **schedule** | {none, cosine}  | **cosine** | 0.486 | 0.489 | 0.455 |
| 4 | **weight decay** | {0, 1e-5, 1e-4, 1e-3} | **0** | **0.486** | **0.489** | **0.455** |

**საუკეთესო SmallCNN მოდელი**
```
SmallCNN(dropout=0.3), norm="none", lr=3e-4, lr_schedule="cosine",
weight_decay=0, 35 epochs, Adam, class weights, no augmentation
→ val 0.486 / test 0.489 / F1 0.455
```

## არქიტექტურა 3 — DeepCNN (კონვოლუციური ბლოკები + BatchNorm + GAP) `deep_cnn.ipynb`

სამი კონვოლუციური ბლოკი - **32 → 64 → 128**. თითო ბლოკი ორი `Conv3×3 → BatchNorm → ReLU` ფენაა,
შემდეგ `MaxPool2d(2)` (48 → 24 → 12 → 6). კლასიფიკატორი არის **GAP → dropout →
`Linear(128, 7)`. პარამეტრების რაოდენობა **~288k** (კვლავ MLP-ზე ნაკლები).

**Baseline** მოდელი (lr 3e-4, cosine, dropout 0.5, 30 epoch):

| model | params | best val | test acc | test F1 | train acc |
|---|---|---|---|---|---|
| SmallCNN (tuned) | 129k | 0.486 | 0.489 | 0.455 | ~0.61 |
| **DeepCNN (baseline)** | 288k | **0.600** | **0.608** | **0.605** | ~0.78 |

ბეიზლაინ DeepCNN მოდელმა აჯობა tuned SmallCNN მოდელს. ტესტზე შედეგი `0.119`-ით უკეთესი, F1 - `0.15`-ით.
თუმცა მოდელი ისევ ოვერფიტში მიდის (გეპი `~0.18`), შესაბამისად სჭირდება რეგულარიზაცია.

### DeepCNN-ის tuning 

გამოვიყენე `grid_search`
| search | პარამეტრი | grid | საუკეთესო | best val | test acc | test F1 |
|---|---|---|---|---|---|---|
| — | baseline | lr 3e-4, 30 ep | — | 0.600 | 0.608 | 0.605 |
| 1 | **learning rate** | {1e-4, 3e-4, 1e-3, 3e-3} | **1e-3** | 0.616 | 0.615 | 0.607 |
| 2 | **dropout** | {0.3, 0.5, 0.6, 0.7} | **0.5**  | 0.616 | 0.615 | 0.607 |
| 3 | **augmentation** | {off, on} | **on** | 0.631 | 0.648 | 0.627 |
| 4 | **epochs (early stop)** | 75-მდე, patience 15 | **~69** | **0.649** | **0.666** | **0.659** |

DeepCNN მოდელს დავამატე აუგმენტაცია, რომელიც ოდნავ ცვლის დატაპოინტებს ტრეინინგისას (ჰორიზონტალური 
დატრიალება, ფოტოს დახრა, ჩამოჭრა), შესაბამისად მოდელი უფრო კარგად განაზოგადებს და არ წავა ოვერფიტში.
აუგმენტაციას ვტესტავდი 45 ეპოქაზე რამეთუ აუგმენტაციის დროს დაქონვერჯებას მეტი დრო სჭირდება.
აუგმენტაციამ გამოიწვია ოვერფიტის გაქრობა- მის გარეშე ტრეინმა **0.96**-ს მიღწია და ვალიდაციის loss-მა **2.29**-ს
ხოლო როდესაც აუგმენტაცია ჩართულია ტრეინი **0.65**-ზე გაჩერდა, გეპი **~0.02**, და ვალიდაციის loss-იც ნაკლებია - **1.08**.

**ეპოქები early stopping-ით** — ვატრეინებ 75 ეპოქამდე და ვჩერდები როდესაც ვალიდაციის სიზუსტე
დიდად აღარ იცვლება. როდესაც `patience = 12, min_delta = 0.005`,  ტრეინინგი გაჩერდა **41**-ე ეპოქაზე, თუმცა საუკეთესო შედეგიც
ამავე ეპოქაზე ჰქონდა, შესაბამისად min_delta შევამცირე `0.001`-მდე, ხოლო patience გავზარდე `15`.

**საუკეთესო DeepCNN-ის მოდელი**

```
DeepCNN(dropout=0.5), norm="batch", GAP head, lr=1e-3, lr_schedule="cosine",
augmentation ON, ~75 epochs (early stop: patience 15, min_delta 0.001),
Adam, class weights
→ val 0.649 / test 0.666 / F1 0.659
```

## არქიტექტურა 4 — ResNetCNN (residual ბლოკები + BatchNorm + GAP) `resnet_cnn.ipynb`

### ResNet-ის tuning — იგივე search-ზე-მართული coordinate descent

| search | პარამეტრი | grid | საუკეთესო | best val | test acc | test F1 |
|---|---|---|---|---|---|---|
| — | baseline | (24,2), lr 1e-3, cosine, aug | — | **0.635** | **0.661** | **0.647** |
| 1 | **learning rate** | {3e-4, 1e-3, 3e-3} | **1e-3** | 0.646 | 0.658 | 0.638 |
| 2 | **label smoothing** | {0, 0.05, 0.1} | **0.1** | 0.667 | 0.671 | 0.640 |
| 3 | **capacity (W×D)** | base channels {16, 24} × blocks {2, 3} | **(24, 2)** | 0.663 | 0.675 | 0.640 |

**საუკეთესო ResNetCNN-ის მოდელი**

```
  ResNetCNN(channels=24, blocks_per_stage=2, dropout=0.3), norm="batch",
  lr=1e-3, label_smoothing=0.1, lr_schedule="cosine", augmentation ON,
  max 60 epochs (early stop: patience 10, min_delta 0.001), Adam, class weights
  → val 0.680 / test 0.686 / F1 0.660  
```

**შეჯამება**

| არქიტექტურა | პარამეტრების რაოდენობა | test acc | test F1 | 
|---|---|---|---|
| MLP (tuned) | 592k | 0.436 | 0.417 | 
| SmallCNN (tuned) | 129k | 0.489 | 0.455 | 
| DeepCNN (baseline) | 288k | 0.608 | 0.605 | 
| **DeepCNN (tuned)** | 288k | **0.666** | **0.659** | 
| ResNetCNN (baseline) | 1.6M | 0.661 | 0.647 | 
| **ResNetCNN (tuned)** | 1.57M | **0.686** | **0.660** |
